Weekly arXiv Digest — Setup Guide
===================================

What it does
------------
Every Monday at 9 AM UTC, a GitHub Actions workflow:

  1. Queries the arXiv API for papers matching LLM-as-a-Judge keywords
     (searches titles and abstracts for 13 keyword variants).
  2. Deduplicates against existing entries in judge_papers.csv
     (matches on arXiv ID and exact title).
  3. Appends any new papers to the CSV with Section and About left blank
     for you to fill in manually.
  4. Recomputes the UMAP embeddings so new papers appear on the scatter plot.
  5. Commits the updated CSV + data files and pushes to main,
     which triggers a GitHub Pages rebuild automatically.
  6. Sends you an email listing the newly added papers.

You can also trigger it manually from the Actions tab → "Weekly arXiv Ingestion" → Run workflow.


Required setup — email notifications
-------------------------------------
The workflow sends email via SMTP. You need to add four repository secrets:

  Repo → Settings → Secrets and variables → Actions → New repository secret

  MAIL_SERVER    SMTP server hostname     (e.g. smtp.gmail.com)
  MAIL_PORT      SMTP port                (e.g. 465 for SSL, 587 for STARTTLS)
  MAIL_USERNAME  SMTP login               (e.g. your-email@gmail.com)
  MAIL_PASSWORD  SMTP password or app password

For Gmail specifically:
  - Go to https://myaccount.google.com/apppasswords
  - Generate an app password (requires 2FA enabled)
  - Use smtp.gmail.com / port 465 / your Gmail address / the app password

Emails are sent to: pangakis95@gmail.com
(Change this in .github/workflows/ingest.yml if needed.)


If no new papers are found
--------------------------
Nothing happens — no commit, no email, no rebuild.


Changing the search keywords
----------------------------
Edit the SEARCH_KEYWORDS list in scripts/ingest_arxiv.py.
